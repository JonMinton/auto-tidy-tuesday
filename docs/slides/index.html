Part 1: Thinking About Agentic AI

What You Think You Know

  • Most people’s mental model of AI is chatbots circa 2023-24
  • Fast, superficial, stateless, text-in/text-out
  • Useful for drafting, summarising, brainstorming

That mental model is already out of date.

This is the key challenge: the audience likely has chatbot-shaped expectations. We need to unlearn those before the new concepts will land. The shift is from “fast autocomplete” to “slower, deeper, situated reasoning”.

What is Agentic AI?

  • Not just faster chatbots — a different paradigm
  • The AI has a role, a goal, and acts in an iterative loop
  • It doesn’t just talk — it does things in your environment
  • It prioritises deeper reasoning over speed

The rest of Part 1 unpacks the building blocks that make this possible. Each concept is something chatbots lack.

Agent, Prompt, I/O

Chatbot

  • You write a message
  • It writes a reply
  • The chat window is the product

Agentic

  • The agent has a persistent role (system prompt, CLAUDE.md)
  • The prompt shapes behaviour across a whole session
  • The chat window is I/O — an interface, not the point

CLAUDE.md is like onboarding notes for a new team member — it tells the agent about project conventions, structure, and goals.

Environment / Context

Chatbot

  • Only sees what you paste in
  • No awareness of surroundings
  • You are the bottleneck

Agentic

  • Sees your filesystem, codebase, data files
  • Understands project structure
  • The agent is situated in your working environment

This is the difference between describing a room to someone over the phone vs having them stand in it with you.

Workspace

  • The actual project directory: files, folders, configs
  • The agent reads, writes, and navigates it like a collaborator
  • This is what makes it useful rather than just clever

A chatbot gives you text to copy-paste. An agent changes your files directly.

State / History & SOPs / Architecture

  • State: the agent “remembers”1 what’s happened — conversation history, previous decisions
  • History: git log, memory files — persistence across sessions
  • SOPs: your project’s conventions (CLAUDE.md, folder structure, coding style)
  • Architecture: the overall design the agent works within

The agent doesn’t just follow instructions — it learns your project’s way of working.

  1. Not true memory — more like handing over detailed notes to someone with a very similar (even identical) set of expertise. Each session is a new agent reading the previous agent’s notes.

Tooling

Chatbot

  • Generates text
  • That’s it

Agentic

  • Read and write files
  • Run shell commands
  • Search codebases
  • Execute code (R, Python, …)
  • Web search, API calls
  • MCP integrations

Tools are what let reasoning become action.

Version Control

  • Git as a safety net: every change is trackable and reversible
  • Enables fearless iteration — try things, revert if wrong
  • The audit trail that makes agentic work reproducible and trustworthy

Without version control, agentic AI would be too risky to use on real projects.

Taste the Cake, Change the Recipe

  • The recipe = source files (.qmd, .R, .py, config)
  • The cake = rendered output (HTML slides, PDF reports, plots)
  • Version control tracks the recipe. The cake is regenerated on demand.

The academic anti-pattern: editing the cake directly — cutting bits of Word doc, pasting into another Word doc, emailing “final_v3_FINAL.docx”

That’s frankencake. It’s fragile, unreproducible, and untraceable.

This is the key habit shift. Most knowledge workers instinctively work on outputs rather than sources. Everything that follows — Quarto, literate programming, agentic workflows — depends on working at the recipe level.

Quarto: An Opinionated Solution

  • Literate programming: prose, code, and output in one source file (.qmd)
  • One recipe, many cakes:
    • These slides → RevealJS (format: revealjs)
    • A paper → PDF or Word (format: pdf, format: docx)
    • A website → HTML pages (type: website)
    • A dashboard → interactive app (format: dashboard)

The source stays the same. The output format is just a setting.

This very presentation is a .qmd file. The sister projects use the same approach for reports and analyses. Quarto is the recipe book; the format is which tin you pour it into.

Why Should Academics Care?

  • Research is inherently iterative and exploratory
  • Data analysis involves many small decisions
  • Reproducibility benefits from explicit, logged reasoning
  • Agentic workflows make the implicit explicit

Complementary Strengths

Humans AI
Visual pattern recognition Gestalt, context, meaning Pixel-level detail, consistency
Domain knowledge Deep, tacit, contextual Broad, explicit, fast retrieval
Iteration speed Slow but purposeful Fast but needs direction
Judgement Nuanced, value-laden Systematic, reproducible

Language Agnosticism

  • R and Python both have strengths
  • R: tidyverse, ggplot2, statistical modelling
  • Python: GIS (geopandas, folium), ML ecosystems
  • Agentic AI can work across both — the reasoning transfers

Part 3: Live Demo — Tidy Tuesday

What is Tidy Tuesday?

  • A weekly social data project — running since April 2018
  • Each week: a new dataset, a data dictionary, and an article for context
  • Community shares visualisations and code using #TidyTuesday
  • ~350+ datasets published across 8 years
  • Originally R-focused, now welcomes R, Python, and Julia

Accessing Tidy Tuesday

Website tidytues.day
GitHub repo github.com/rfordatascience/tidytuesday
R package install.packages("tidytuesdayR")
Load a dataset tidytuesdayR::tt_load("2025-08-19")
Community Data Science Learning Community (DSLC)

Used in 30+ university courses worldwide as a teaching resource.

How This Works

  1. You choose a dataset from the list
  2. You choose an output format:
    • Slide deck (RevealJS) — for a presentation
    • Scientific notebook (HTML) — for exploratory analysis
    • Academic PDF — with references and structured sections
    • Interactive website — with live tables and maps
  3. We explore it together with an AI agent
  4. Watch the iterative process unfold

Same .qmd source, different format: setting. One recipe, your choice of cake.

What to Watch For

  • How the agent asks clarifying questions
  • How it chooses between R and Python
  • How human judgement steers the analysis
  • Where the AI’s visual reasoning complements ours
  • Where it falls short and needs guidance
